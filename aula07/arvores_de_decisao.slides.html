Introduçao¶

  • rvores de Deciso (AD) so uma das tcnicas mais populares de minerao de dados.

  • As AD consistem em um conjunto de ns internos e ns folhas que esto organizados hierrquicamente.

  • Essa estrutura permite organizar o processo de deciso.
  • Um exemplo, poderia ser uma rvore de deciso para decidir se ir jogar tnis ou no, baseado em alguns atributos climticos.

  • Algumas vantagens das rvores de deciso:

    • O conhecimento explcito e interpretvel.

    • O processo de tomada de deciso , alm de entendvel, explicvel.

    • Podem ser representados por meio de smbolos usando a forma se/ento, por isso chamado de modelo simblico.

Principais Algoritmos¶

  • Os algoritmos de induo de classificadores que representam o conhecimento adquirido por meio de uma rvores de Deciso, pertencem a uma famlia de algoritmos chamados TDIDT (Top Down Induction of Decision Trees).
  • Existe uma literatura extensa sobre algoritmos clssicos e adaptades de algoritmos, mas alguns dos principais:
    • ID3 (Iterative Dichotomiser 3)
      • Proposto por Quinlan em 1986
      • Suporta atributos discretos e classe tambm discreta.
    • C4.5
      • Proposto por Quinlan em 1993
      • Suporta atributos discretos e contnuos, e classe discreta.
    • M5
      • Proposto por Quinlan em 1992
      • Suporta atributos discretos e contnuos, e classe numrica
      • Neste caso, chamam-se rvores de regresso

Algoritmo Geral¶

  • A idia geral do algoritmo consiste em dividir o conjunto de dados sucessivamente at que todos ns folha sejam puros.
  • Mtodo $AD(X)$
    1. Se todos os exemplos em $X$ so da mesma classe cria-se uma n folha para essa classe;
    2. Caso contrrio:
      1. Mede-se a qualidade de todos os atributos;
      2. Se nenhum atributo consegue melhorar a organizao do conjunto $X$ ento cria-se um n folha para a classe mais frequnte;
      3. Caso contrrio:
        1. Escolhe o melhor atributo $A$ para $X$;
        2. Cria-se um n interno de deciso baseado em $A$;
        3. Divide-se o conjunto de dados de acordo com os valores de $A$ em subconjuntos $X_j$;
        4. A partir do n de deciso cria-se uma aresta para cada valor de $A$ e associa-se cada $X_j$ a uma aresta;
        5. Para cada conjunto $X_j$ executa-se o mtodo $AD(X_j)$;

Algoritmos ID3 e J48¶

  • A complexidade desses algoritmos da ordem $O(n \times m^2)$, onde $n$ o nmero de exemplos e $m$ o nmero de atributos.

  • Os algoritmos so gulosos (greedy), pois a cada iterao eles escolhem um atributo de forma gulosa, baseado numa medida importncia de atributos, que s olha para o estado atual do problema.

  • O algoritmo no timo, pois no explora todas as possveis rvores para escolher a melhor.

  • O algoritmo completo, pois sempre contri uma rvore, nem que seja apenas um n raiz folha que para a classe majoritria.
